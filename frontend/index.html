Backend Developer: Assignment

There are two modules which work as follows:

1. User Management Module: Allows admin to add, delete, modify a user and map his work area by defining city and locality through drop down selection. The drop-down data comes through APIs from the second module â€“ Monitored Geo Module

2. Monitored Geo Module allows admin to add/delete new city and new locality. It also allows association of a locality to a city.

Functionalities

Users Option Contains All the Users with respective Edit(Working as per requirements) and Delete Option(Working as per requirements), in the header part there is an option of Add User. In Add User the user first have to select the city. and then corresponding localities is shown. Also in the Update User Page the id remains fixed we can change everything.

Cities Option Contains All the Cities with respective Edit(Not working) and Delete Option(as per requirements), in the header part there is an option of Add City.

Localities Option Contains All the Localities with respective Edit(Not working) and Delete Option(as per requirements), in the header part there is an option of Add Locality. In Add Locality the user must have to select the city.

Assumptions

By Default the User is Admin. Firstly Admin needs to add City, then the Admin can add Locality through drop down selection for city Only those Locality is shown to Admin on the basis of the Selected city. Admin can add the User Admin can map the Work firstly by Selecting the City and thereby Locality.
